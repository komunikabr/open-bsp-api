API Keys

Passe a API Key no header Authorization: Bearer <KEY> em todas as requisições à nossa API.

Endpoints da API

Integre diretamente com nossa REST API — sem se preocupar com a infraestrutura

Base URL v1
https://api.msghub.com.br/v1
Mensagens
POST /messages/send Enviar mensagem de texto ou template
GET /messages Listar mensagens de uma conversa
Conversas
GET /conversations Listar conversas ativas
GET /conversations/:id Detalhes de uma conversa
Contatos
GET /contacts Buscar contatos por nome ou número
POST /contacts Criar ou atualizar contato
Templates
GET /templates Listar templates aprovados
Exemplo — Enviar mensagem 
const response = await fetch('https://api.msghub.com.br/v1/messages/send', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${''}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    to: '5511999998888',
    message: { text: 'Olá! Como posso ajudar?' }
  })
})

const data = await response.json()
console.log(data)
import requests

response = requests.post(
    'https://api.msghub.com.br/v1/messages/send',
    headers={
        'Authorization': 'Bearer <SUA_API_KEY>',
        'Content-Type': 'application/json'
    },
    json={
        'to': '5511999998888',
        'message': {'text': 'Olá! Como posso ajudar?'}
    }
)
print(response.json())
curl -X POST https://api.msghub.com.br/v1/messages/send \
  -H "Authorization: Bearer <SUA_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "5511999998888",
    "message": {"text": "Olá! Como posso ajudar?"}
  }'

Webhooks

Receba eventos em tempo real na URL da sua aplicação

Como funciona

Cliente envia mensagem WA
MsgHub processa e roteia
POST para sua URL
Sua app responde

Payload enviado para sua URL:

{
  "event": "message.received",
  "app_id": "seu-app-id",
  "data": {
    "from": "5511999998888",
    "text": "Olá, preciso de ajuda!",
    "timestamp": "2025-01-15T14:30:00Z"
  }
}
